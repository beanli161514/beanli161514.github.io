Basic Representation of Crystalline Material and Some popular Networks

Posted on 2022-02-26 | In Graduate Work

When predicting properties of crystal materials, the proper the main idea in existing methods based on convolutional graph neural nerwork is to present the structure by a crystal graph that encodes both atomic information and bonding interactions between atoms. The encoding technique should be able to represent as more information of the materials as possible, while ensuring that the data is optimum for networks to converge. For CNN based methods, the main idea is to represent the crystal as a 2D vector, i.e. an image. It make usage of the saptial position of atoms on the periodic table and the quantity of atoms in a crystal. The representation isself does not contain any connection information of the crystal, during training the network learns part of the connection information automatically. However, this CNN based method only works well on datasets which contain only one type of material, for instance, ABX2 crystal. I guess this limitation results form it’s inability to determine various connections only from atomic formula.

CGCNN

CGCNN is the first approach using GCN to address material properity prediction problem.

As illustrated in the paper, a crystal graph $\mathcal{G}$ is an undirected multigraph which is defined buy nodes representing atoms and edges representing connections between atoms in a crystal. The crystal graph is unlike normal graphs since it allows multiple edges between the same pair of end nodes, a characteristic for crystal graphs due their periodicity. Each node $i$ is represented by a feature vector $v_i$, encoding the property of the atom corresponding to node $i$, which is determined by the orbital electrons of the atom. Similarly, each edge $(i,j)k$ is represented by a feature vector $u{(i,j)_k}$ corresponding to the $k$th bond connecting atom $i$ and atom $j$, which is a kernelized distance features.

The convolutional layers iteratively update the atom feature vector $v_i$ by convolution with surrounding atoms and bonds with a nonlinear convolution function.

$$\boldsymbol{v}{i}^{(t+1)}=\boldsymbol{v}{i}^{(t)}+\sum_{j, k} \sigma\left(\boldsymbol{z}{(i, j){k}}^{(t)} \boldsymbol{W}{f}^{(t)}+\boldsymbol{b}{f}^{(t)}\right)\odot g\left(\boldsymbol{z}{(i, j){k}}^{(t)} \boldsymbol{W}{s}^{(t)}+\boldsymbol{b}{s}^{(t)}\right)$$

After $R$ convolutions, the network automatically learns the learns the feature vector $v_i^{(R)}$ for each atom by iteratively including its surrounding environment. The pooling layer is then used for producing an overall feature vector $v_c$ for the crystal, which can be represented by a pooling function

$$\boldsymbol{v}{\boldsymbol{c}}=\operatorname{Pool}\left(\boldsymbol{v}{0}^{(0)}, \boldsymbol{v}{1}^{(0)}, \ldots, \boldsymbol{v}{N}^{(0)}, \ldots, \boldsymbol{v}_{N}^{(R)}\right)$$

OCGNN

OCGNN is abbreviation of Orbital Graph Convolutional Neural Network. Compairing with CGCNN, it introduces orbital field matrix that encodes orbital interactions according to electron conﬁgurations of the central atom and neighbor atoms surrounding the central atom as a 2D vector and then flatten the feature vector to adapt CGCNN network.

In this work, the electron configuration of each atom is converted into a one-dimensional binary vector, and the local structure surrounding an atom is encoded into a matrix that is the sum of the weightd vector reprentation of all neighboring atoms. For a center atom $c$, assuming that it has $M$ nearest neighbors, denoted as $n$, and the electron configurations of them are encoded into 1D binary vectors $O_c$ and $O_n$. To get the OFM, an weight representing the bond strength between atoms is calculated by $w_{c n}\left(r_{c n}\right)=\theta_{c n} \times \zeta\left(r_{c n}\right)$. Where $\zeta\left(r_{c n}\right)$ is a function of the distance between atoms, $\theta_{c n}$ denotes the solid angle between center-neighbor pairs in the Voronoi cell. To construct the OFM for the center atom, the authers sum the matrix product for each center-neighbor pair and multiply it with the corresponding weight function. This results in a 32 $\times$ 32 matrix.

$$X_{c}=\vec{O}{c}^T+\sum{n=1}^{M} \vec{O}{c}^T \vec{O}{n} \theta_{c n} \zeta\left(r_{c n}\right)$$

The authors concatenate the atom orbital vector directly to the OFM, which is a little confusing for me. I will make some change to this in my work.

Then, another unproper process was conducted. The OFM was flattened into a 1D vector and concatenated with the atom feature vector of the center atom. Then, this 1148 $\times$ 1 vector was fed into a stupiedly large GCN inherited from CGCNN.

My version of OCGNN

$$X_{c}=\sum_{n=0}^{M} \vec{O}{c}^T \vec{O}{n} \theta_{c n} \zeta\left(r_{c n}\right)$$

This version of OFM doesn’t contain $X_{c}=\vec{O}_{c}^T$. Instead, I use $\vec{O}_c^T \vec{O}_c$.

Then a CNN is used to extract features from OFM. After several convolution layers, the feature of OFM is a 1D vector, which is suitable for GCN or graph attention based methods.
